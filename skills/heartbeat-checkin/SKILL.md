# 心跳簽到

## 概述

每 2 小時向 Telegram 發送一個引導性問句，作為神經多樣性的外部鷹架，幫助維持專注節奏、避免漂移。

## 步驟

### 1. 判斷當前時段

根據系統時間判斷情境標籤：

| 時段 | 標籤 |
|------|------|
| 06:00–09:00 | 早晨啟動 |
| 09:00–12:00 | 上午衝刺 |
| 12:00–14:00 | 午間緩衝 |
| 14:00–17:00 | 下午主力 |
| 17:00–19:00 | 收尾段 |
| 19:00–22:00 | 晚間沉澱 |
| 22:00–06:00 | 深夜模式 |

```bash
date +"%H:%M"
```

### 2. 從問題池隨機選句

從以下問句池中隨機選 1 句：

**專注類**
- 「現在在做什麼？還是卡住了？」
- 「上一個小時有沒有完成一件事？」
- 「現在最重要的一件事是什麼？」

**狀態類**
- 「身體怎麼樣？要不要喝水或站起來？」
- 「腦袋清醒嗎？還是開始打轉？」
- 「現在的能量值大概幾成？」

**推進類**
- 「有沒有一件小事可以在 15 分鐘內完成？」
- 「今天有沒有已經完成的事，可以先記下來？」
- 「卡在哪裡？說出來看看。」

**收尾類**（17:00 後適用）
- 「今天完成最重要的一件事了嗎？」
- 「有沒有什麼事要在明天繼續，現在記一下？」
- 「幾點打算收工？」

### 3. 組合訊息

格式：
```
[時段標籤] HH:MM 心跳 💚

{問句}
```

範例：
```
[上午衝刺] 10:00 心跳 💚

現在在做什麼？還是卡住了？
```

### 4. 透過 Telegram 發送

使用 `mcp__plugin_telegram_telegram__reply` 發送到用戶的 Telegram。

- `chat_id`：從 MEMORY.md 或上下文取得
- 不需要 `reply_to`（主動發送，不是回覆）
- 不加多餘說明，只發問句本身

## 備註

- **不要等回覆再繼續**：這是單向發送的鷹架觸發，不是對話啟動
- **深夜模式安靜**：22:00–06:00 預設不發送，除非用戶明確要求
- **問句要短**：最多 2 句，不解釋、不說教
- **手動觸發**：用戶說「簽到」「心跳」「現在狀態」時，執行一次本流程
- **定時觸發**：可搭配 `/schedule` 設定每 2 小時自動執行（需用戶另行設定）
