# UNIQLO — Playwright 食譜

**最後更新：** 2026-04-05
**測試狀態：** 🚧 踩坑記錄中
**入口 URL：** https://www.uniqlo.com/tw/

## 關鍵限制（先讀）

- **有頭模式才能用**：UNIQLO 偵測 headless，會擋掉或顯示異常。playwright 已改為預設 headed。
- **兩段式查詢**：第一段查商品/庫存，第二段確認尺碼/顏色細節，一次不能塞太多操作。
- **需要用戶 Chrome cookie**：如果要看購物車狀態或會員折扣，需要使用者手動登入一次；純查詢不需要。

## 操作流程

### 查詢商品庫存

**第一段（商品搜尋）：**
1. 前往 `https://www.uniqlo.com/tw/`
2. 使用搜尋欄輸入商品名稱
3. 等待搜尋結果載入（wait 2 秒）
4. 截圖確認結果頁，取商品連結

**第二段（確認細節）：**
1. 前往商品頁面
2. 等待頁面完整載入（wait 2 秒）
3. 截圖取得尺碼 × 顏色庫存矩陣
4. 回報格式：
   ```
   UNIQLO [商品名稱]
   顏色：[顏色A] [顏色B]
   尺碼庫存：
     S：[顏色A]✅ [顏色B]❌
     M：[顏色A]✅ [顏色B]✅
   ```

## 已知的坑

- headless 模式被擋，一律用 headed（已在 .mcp.json 設定）
- 庫存狀態是 JS 渲染，截圖前需等 2 秒
- 如果被 CAPTCHA 擋，截圖回報，人工處理
- 購物車/結帳需要登入 cookie，純查詢不需要
- **分段設計原則**：搜尋任務 token 消耗大，每段盡量輕量。實際切割粒度視任務複雜度決定，沒有固定公式，跑過才知道。
