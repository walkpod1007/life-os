# Life OS 起步指南：用 Claude Code 建立個人記憶

> 產出：Opus，2026-03-22
> 目的：讓你在 Claude Code 裡開始累積「關於你」的資料
> 原則：不開發，只使用。每天五分鐘，語音或打字都行。

---

## 你現在有什麼

| 工具 | 角色 | 狀態 |
|------|------|------|
| Claude Code + CLAUDE.local.md | 個人認知夥伴，認識你 | 今天開始 |
| 龍蝦（LINE） | 工作群組互動 + 生活遙控器 | 繼續用，Push 額度恢復後再擴展 |
| GPT Codex | 一次性腳本 + 自動化 | 需要時才開 |
| Google Calendar | 固定排程提醒 | 已在用 |
| Obsidian Vault | 知識管理 | 需要分三區整理 |

---

## Vault 三區分離（Obsidian）

```
你的 Vault/
├── 🧠 人類/              ← 你的筆記、日記、想法
├── 🤝 人類+龍蝦/          ← 生活資料（車、健康、消耗品、財務）
│   └── 龍蝦可讀可寫，你也可以看
└── 🦞 龍蝦系統/           ← SOUL.md、AGENTS.md、Skills、技術設定
    └── 你平常不用看，想確認時再開
```

先建好這三個資料夾，把現有的文件搬進去。不用一次搬完，看到什麼就分類什麼。

## 第一週：教 Claude Code 認識你

每天開一次 Claude Code session，花五分鐘講一個主題。
講完後叫它更新 CLAUDE.local.md。

### Day 1：車子
> 「我有一台 ___（廠牌型號年份），目前里程大約 ___ 公里。
> 上次保養是 ___ 月，在 ___（保養廠）。
> 通常每 ___ 公里或 ___ 個月保養一次。」

### Day 2：健康
> 「我的牙醫在 ___，上次洗牙是 ___ 月。
> 我固定在吃的藥是 ___。
> 上次健康檢查是 ___ 年 ___ 月。」

### Day 3：財務週期
> 「房租每月 ___ 號繳，金額 ___。
> 車險 ___ 月到期。
> 其他固定支出：___。」

### Day 4：消耗品
> 「口罩一次買 ___ 片，大概用 ___ 個月。
> 其他定期要補的東西：___。」

### Day 5：生活節奏
> 「我希望每天 ___ 點洗衣服、___ 點折衣服。
> 每週日 ___ 點剪指甲。
> 我的精神狀態通常週 ___ 最好、週 ___ 最差。」

### Day 6：工作節奏
> 「每天大概 ___ 點到辦公室、___ 點離開。
> 最常被打斷的事情是 ___。
> 最希望自動化的重複工作是 ___。」

### Day 7：回顧
> 「讀一下 CLAUDE.local.md，告訴我你現在對我的理解。
> 有沒有什麼地方不對或不完整的？」

## 之後的日常：隨手告訴 Claude Code

不用刻意，遇到就說：

- 「今天換了機油，里程 42000」→ Claude Code 更新 CLAUDE.local.md
- 「剛看完牙醫，下次回診 7 月」→ 更新健康區塊
- 「口罩剛買了一箱」→ 更新消耗品，重算下次補貨時間
- 「這個月 A 客戶的款項延遲了」→ 記進 memory

每次花不到 10 秒。累積三個月，Claude Code 就會比任何人都了解你的生活節奏。

---

## 什麼時候把 Life OS 接回龍蝦

等以下條件都滿足時：

1. ✅ CLAUDE.local.md 已經有你的完整生活資料
2. ✅ 你已經習慣「隨手告訴 AI」這個動作
3. ✅ 龍蝦的 Push 額度恢復
4. ✅ 龍蝦有「收到生活資訊 → 存檔」的 Skill（`life-context`）

到時候把 CLAUDE.local.md 的內容同步到龍蝦的 USER.md 或 MEMORY.md，它就能在 LINE 上做定時提醒、主動推播、幫你回答別人的查詢。

在那之前，Claude Code 負責認識你，龍蝦負責工作。兩邊不衝突。

## 最重要的一件事

**今天就開始 Day 1。不要等「準備好了」才開始。**

打開 Claude Code，說一句：「我的車是 ___，上次保養是 ___ 月。」

就這樣。五秒鐘。剩下的明天再說。
