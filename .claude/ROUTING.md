# 三大金剛派工指南

> 自動化任務路由：Gemini CLI、Codex CLI、Claude Code

## 快速判斷（選誰出戰）

| 任務類型 | 工具 | 例子 | 速度 |
|---------|------|------|------|
| **One-shot 內容** | Gemini CLI | 摘要、改寫、清單、JSON輸出 | ⚡ 快 |
| **工程分析/修改** | Codex CLI | 代碼分析、diff生成、review、單檔修改 | ⚡⚡ 中快 |
| **長流程工程** | Claude Code | 跨檔重構、改碼→跑測試→迭代、複雜架構 | 🏗️ 慢但完整 |

## 派工邏輯

### 🎬 Gemini CLI（內容工人）
**何時用**：
- YouTube/文章摘要
- 文本改寫、翻譯
- JSON 生成（清單、結構化數據）
- 快速查詢、點子激發

**命令範式**：
```bash
gemini "prompt" [URL]
gemini --output-format json "prompt"
```

**成本**：便宜、快速、無副作用

---

### 🔧 Codex CLI（工程助手）
**何時用**：
- 單個檔案分析（邏輯、bug、改進點）
- 小規模修改（幾行代碼、配置調整）
- 代碼 review
- 工程決策諮詢（架構、設計模式）

**命令範式**：
```bash
codex exec -C <workdir> "prompt"
codex -C <workdir> --no-alt-screen  # 互動模式
```

**成本**：中等、適合一次性任務

---

### 🏗️ Claude Code（我自己）
**何時用**：
- 跨多檔修改（重構、新功能）
- 改碼 → 跑測試 → 迭代
- 複雜架構設計、多步驟任務
- 需要對話反饋的工程

**特性**：
- 全上下文（能記住你的代碼、決策、偏好）
- 持久化（可以跨 session 記住進度）
- 互動式（可以邊做邊調整）

**成本**：貴、但完整

```
有工作嗎？
├─ 是內容工作（摘要、改寫、查詢）

```

## 實戰案例

### ✅ YouTube 摘要 → Gemini
```
用戶：「https://youtu.be/xxx 丟給 Gemini 出 800 字摘要」
我：gemini "請總結這個 YouTube 影片，800字繁體中文摘要" URL
結果：⚡ 30秒內返回
```

### ✅ 檔案分析 → Codex
```
用戶：「分析 compose-poster.py 的縮放邏輯」
我：codex exec -C /path/to/scripts/ "分析 compose-poster.py..."
結果：⚡ 代碼行號、修改建議
```

### ✅ 複雜重構 → Claude Code
```
用戶：「重構整個 Keep 同步系統，新增語意搜尋」
我：（你就在跟我對話，不派工）
結果：🏗️ 跨檔修改 + 測試 + 迭代
```

## 派工 SLA

| 工具 | 預期時間 | 失敗率 | 適合場景 |
|------|---------|--------|---------|
| Gemini | <1min | 1% | 急、小、簡單 |
| Codex | 1-3min | 5% | 中等複雜度 |
| Claude Code | 5min+ | <1% | 複雜、需反饋 |

---

## 禁忌

❌ **不要做**：
- 把 Gemini 當萬能工具（工程活留給 Codex/我）
- 跨檔工作派給 Codex（它只看一個檔案）
- 簡單摘要派給 Claude Code（燒 token）
- 派工時說「幫我做」，沒說是什麼工作

✅ **要做**：
- 明確說任務類型
- 提供工作目錄/檔案路徑
- 派工時帶上 context（背景、目標、限制）
